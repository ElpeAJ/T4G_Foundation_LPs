  • Lesson 3: Overview

    This lesson introduces you to the Web development, abbreviations, extentions, file and folder structure, installation on Web development tools, and signing up for cloud and hosting applications

    Tasks:

    1. 1. Download and install IDE (VSCode).
    2. 2. Download and install Git.
    3. 3. Sign up for GitHub.
    4. 4. Create a T4G folder on the Desktop.
    5. 5. Create 6 folders inside the T4G folder (lesson1 to lesson6).
    6. 6. Create folders inside each lesson's folder: pages folder, src folder and images folder
    7. 7. Create different files with different file extensions: index.html file, style.css
    8. 8. Using the Doctype shortcut, Learn the structure of an HTML file and open it in the browser.
    9. 9. Create a GitHub Repository for lesson1 & lesson2
    10. 10. Push lesson1 and lesson2 folders to GitHub
    11. 11. Change the title of the index.html file of lesson1 and lesson2 and push changes to GitHub

      • i. an about us file

    7. B. an src folder - each should contain:
      • i. a style.css file

    11. Deploy each Repository using GitHub pages

  • Lesson 4: Overview

    This lesson covers the syntax for writing HTML code and how it creates the layout of every webpage in a website. It also touches using git commands to push code on GitHub and deployment.

    Tasks

    1. 1. Learn the syntax of HTML.
    2. 2. Create 6 headings using the six different heading tags.
    3. 3. Create paragraphs using p tags and break a paragraph using the break tag.
    4. 4. Inside the p elements use bold tags, italics tag, underline tag, etc.
    5. 6. View History or Status of files using "git status"
    6. 7. Stage changes for a specific file using "git add --filename--" or stage all changes in the current folder/directory using "git add ."
    7. 8. Commit staged changes with a descriptive message using "git commit -m"Message" "
    8. 9. Push local changes to the remote repository using "git push".
    9. 10. View the remote URLs linked to your repository using "git remote -v".
